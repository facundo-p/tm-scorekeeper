## Spacing Scale

Declared values — all sourced from existing `index.css` tokens. No new tokens needed.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--spacing-xs`) | Icon gaps, inline padding, leaderboard cell insets |
| sm | 8px (`--spacing-sm`) | Compact row gaps, legend item spacing |
| md | 16px (`--spacing-md`) | Default element spacing, tooltip padding |
| lg | 24px (`--spacing-lg`) | Section padding, chart container padding |
| xl | 32px (`--spacing-xl`) | Layout gaps between chart and leaderboard |
| 2xl | 48px (`--spacing-2xl`) | Major section breaks |

Exceptions:
- Chart container height: 280px on mobile, 400px on desktop (≥768px) — these are not spacing values but fixed SVG viewport heights set on the wrapping div (12-CONTEXT.md D-07)
- Touch targets: buttons retain `min-height: 44px` (existing Button pattern)
- Leaderboard rows: `min-height: 44px` to comply with touch target minimums on mobile

Source: `frontend/src/index.css` (pre-populated); heights from CONTEXT.md D-07

## Typography

All sizes map to existing `index.css` CSS variables. No new font sizes or weights are introduced.

| Role | Size | Weight | Line Height | CSS Token |
|------|------|--------|-------------|-----------|
| Body | 16px (1rem) | 400 (`--font-weight-normal`) | 1.5 | `--font-size-base` |
| Label / small | 14px (0.875rem) | 400 (`--font-weight-normal`) | 1.5 | `--font-size-sm` |
| Table header | 14px (0.875rem) | 600 (`--font-weight-semibold`) | 1.3 | `--font-size-sm` + `--font-weight-semibold` |
| Section heading | 18px (1.125rem) | 600 (`--font-weight-semibold`) | 1.3 | `--font-size-lg` + `--font-weight-semibold` |

**Chart-specific typography:**
- X-axis tick labels: 14px / weight 400 / `--color-text-muted` — short format `DD/MM` (e.g. `15/03`)
- Y-axis tick labels: 14px / weight 400 / `--color-text-muted`
- Tooltip player name: 14px / weight 600 / `--color-text`
- Tooltip date and elo_after values: 14px / weight 400 / `--color-text`
- Leaderboard delta positive: 14px / weight 400 / `--color-success` (`#27ae60`)
- Leaderboard delta negative: 14px / weight 400 / `--color-error` (`#e74c3c`)
- Leaderboard delta zero: 14px / weight 400 / `--color-text-muted`
- Single-point hint: 14px / weight 400 / `--color-text-muted` — "Solo hay una partida en este rango"

Source: `frontend/src/index.css` (pre-populated from existing tokens); EloSummaryCard.module.css delta patterns reused for leaderboard

**ELO line color palette — deterministic, id-keyed:**

All 6 colors verified ≥3:1 WCAG contrast against `--color-surface` (#2c1810):

| Index | Hex | Approx. contrast vs #2c1810 | Name |
|-------|-----|-----------------------------|------|
| 0 | `#4e9af1` | 4.8:1 | Bright blue |
| 1 | `#f1c40f` | 8.2:1 | Yellow |
| 2 | `#2ecc71` | 5.6:1 | Green |
| 3 | `#e91e8c` | 4.3:1 | Pink |
| 4 | `#ff7043` | 4.1:1 | Orange-red |
| 5 | `#a78bfa` | 4.5:1 | Purple |

Assignment: `palette[hashPlayerId(player_id) % 6]` — hash = sum of UTF-16 char codes of `player_id` string. Deterministic across renders; adding/removing players never shifts existing assignments.

## Interaction Contract

This section is specific to this phase's interactive components.

### Chart: EloLineChart

| Property | Value | Source |
|----------|-------|--------|
| `trigger` | `"click"` everywhere (mobile + desktop) | D-04 |
| `dot` | `false` (default); `true` only when `totalPoints === 1` | D-05, D-06 |
| `isAnimationActive` | `false` | D-05 |
| Y-axis `domain` | `['dataMin - 50', 'dataMax + 50']` | D-05 |
| `cursor` | visible cursor line (Recharts default cursor on click trigger) | D-04 |
| Chart container height — mobile | 280px (set on wrapping `div`, `ResponsiveContainer` handles width) | D-07 |
| Chart container height — desktop | 400px at `min-width: 768px` via CSS Media Query in `EloLineChart.module.css` | D-07 |
| Legend | Recharts `<Legend>` inside chart, color swatch + player name | D-02 |
| X-axis tick format | `DD/MM` short format (e.g. `15/03`) — sparse enough for mobile | Claude's Discretion |

**Tooltip content (per data point on click):**
```
[Player Name]
Fecha: YYYY-MM-DD
ELO: 1523
```
No date reformatting — display the raw `YYYY-MM-DD` string from the DTO (D-04).

### Single-point edge case

When `totalPoints === 1` across all filtered players:
- Override `dot={true}` on the `<Line>` with the data point
- Render a muted caption **below the chart**, outside `<EloLineChart>`: "Solo hay una partida en este rango"
- Caption lives in `Ranking.tsx`, conditional on `totalPoints === 1`
- Caption is not inside the chart component

Source: D-06

### Leaderboard: EloLeaderboard

| Property | Value | Source |
|----------|-------|--------|
| Data source | `dataset` (full unfiltered history) — NOT `filtered` | D-08 |
| Sort | ELO descending; tiebreaker: `player_name` alphabetical (stable) | D-09 |
| "Posición" | 1-based rank after sort | D-09 |
| "ELO actual" | Last `elo_after` from player's full history | D-09 |
| "Última delta" | Last `delta` from player's full history (global, ignores date filter) | D-08 |
| Delta display | `+N` green / `-N` red / `±0` muted — same pattern as `EloSummaryCard` | EloSummaryCard.module.css pattern |
| Row min-height | 44px — touch target compliance | Mobile-first constraint |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Chart wrapper | `role="img"` + `aria-label="Gráfico de evolución de ELO por jugador"` on the outer `div` |
| Recharts accessibility | `accessibilityLayer` prop enabled on `<LineChart>` |
| Screen reader fallback | `<details><summary>Ver datos como tabla</summary>` collapsible table with all visible data points, below the chart |
| CSS isolation | Zero `.recharts-*` selectors in `index.css`; all Recharts styling via CSS Module `className` props |

## Component File Structure

```
frontend/src/components/
  EloLineChart/
    EloLineChart.tsx          ← chart component; receives `data: PlayerEloHistoryDTO[]`
    EloLineChart.module.css   ← container height (280px mobile / 400px desktop), isolation
  EloLeaderboard/
    EloLeaderboard.tsx        ← table component; receives `dataset: PlayerEloHistoryDTO[]`
    EloLeaderboard.module.css ← table layout, delta colors (reuse --color-success/error tokens)
```

Integration in `Ranking.tsx`:
- Replace `renderChartSkeleton()` call with `<EloLineChart data={filtered} />`
- Add `<EloLeaderboard data={dataset} />` below
- Single-point hint conditional: `{totalPoints === 1 && <p className={styles.singlePointHint}>Solo hay una partida en este rango</p>}` between chart and leaderboard
