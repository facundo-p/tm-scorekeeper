# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom CSS variables in `index.css` |
| Preset | not applicable |
| Component library | none (hand-rolled, React functional + CSS Modules) |
| Icon library | lucide-react (to be installed — D-21 from CONTEXT.md) |
| Font | System stack: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto |

Styling approach: CSS Modules (`ComponentName.module.css`) per component. All tokens consumed via CSS custom properties defined in `frontend/src/index.css`. No shadcn. No Tailwind.

## Spacing Scale

All tokens pre-populated from `frontend/src/index.css`. No new tokens needed for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--spacing-xs`) | Icon gaps, progress bar padding, inline gaps |
| sm | 8px (`--spacing-sm`) | Card inner gap, row gap between achievement cards |
| md | 16px (`--spacing-md`) | Card horizontal padding, modal body padding |
| lg | 24px (`--spacing-lg`) | Section gap, tab content padding |
| xl | 32px (`--spacing-xl`) | Empty-state vertical padding |
| 2xl | 48px (`--spacing-2xl`) | Modal top/bottom padding on large screens |

Exceptions:
- Achievement icon circle diameter: 48px (not a gap — a fixed dimension, multiple of 8)
- Tab bar height: 44px (touch target minimum, per WCAG 2.5.5)
- Progress bar height: 6px (visual element, not spacing — follows design reference)

## Typography

All tokens pre-populated from `frontend/src/index.css`. Source: `index.css`.

| Role | Size | Weight | Line Height | CSS Token |
|------|------|--------|-------------|-----------|
| Body | 16px (1rem) | 400 normal | 1.5 | `--font-size-base` / `--font-weight-normal` |
| Label / Caption | 14px (0.875rem) | 600 semibold | 1.3 | `--font-size-sm` / `--font-weight-semibold` |
| Heading (section) | 18px (1.125rem) | 600 semibold | 1.3 | `--font-size-lg` / `--font-weight-semibold` |
| Achievement title | 16px (1rem) | 600 semibold | 1.3 | `--font-size-base` / `--font-weight-semibold` |

Rules:
- LEVEL N label (e.g. "LEVEL 1"): 12px, weight 600 semibold, uppercase, letter-spacing 0.05em. Use `--font-size-sm` scaled down via inline `font-size: 0.75rem` or define `--font-size-xs: 0.75rem` locally in the component module.
- Achievement description: 14px, weight 400, color `--color-text-muted`.
- Tab labels: 14px, weight 600, uppercase, letter-spacing 0.05em.
- Counter (e.g. "2/3"): 12px, weight 600, color `--color-text-muted`.

## Color

All tokens pre-populated from `frontend/src/index.css`. Terraforming Mars dark theme (deep brown/red).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a0a05` (`--color-background`) | Page background, modal overlay target |
| Secondary (30%) | `#2c1810` (`--color-surface`) | Achievement cards, tab bar background, modal background |
| Accent (10%) | `#e67e22` (`--color-accent`) | Reserved — see accent list below |
| Upgrade accent | `#27ae60` (`--color-success`) | "Logro mejorado" border in end-of-game modal only |
| Destructive | `#e74c3c` (`--color-error`) | Not used in this phase |

**Accent (`#e67e22`) is reserved exclusively for:**
1. LEVEL N label text on unlocked achievements
2. Progress bar fill on unlocked achievements
3. New achievement border in the end-of-game modal (new unlock, tier 1)
4. Active tab indicator (bottom border of selected tab)
5. Lucide icon color on unlocked achievement cards

**Grayscale state for locked achievements:**
- Icon: `filter: grayscale(100%)` + `opacity: 0.5`
- LEVEL label: color `--color-text-muted` (`#a89080`)
- Progress bar fill: `--color-border` (`#4a2c1a`) at 0% width
- Card border: `--color-border` (default, no accent)
- Card background: `--color-surface` (no tint)

**Unlocked achievement card:**
- Border: `--color-border` default (no accent border on card — accent is inside the card)
- Background: `--color-surface` (`#2c1810`)

**Note:** Upgrade accent `#27ae60` applies only to the border of upgrade-type mini-badges in the end-of-game modal (D-04 from CONTEXT.md). New unlock uses accent `#e67e22` border. This is the sole visual differentiator between "Nuevo logro" and "Logro mejorado".

## Layout Specifications

### AchievementCard (profile + catalog)

Based on the visual reference (Ahead app screenshot) — card layout:

```
┌─────────────────────────────────────────┐
│  [●icon]  Title (semibold 16px)          │
│  48×48px  Description (muted 14px)       │
│           ──────────────────────────     │
│  LEVEL N  [████████░░░░░░░░░░░░]  2/3   │
└─────────────────────────────────────────┘
```

- Outer container: `flex-direction: row`, gap 12px (`var(--spacing-sm)` + 4px — use 12px as one-off or round to `--spacing-md`; use 12px directly since it falls between tokens). Padding: `--spacing-md` (16px) all sides.
- Icon circle: 48px × 48px, `border-radius: 50%`, background `--color-border` (`#4a2c1a`). Lucide icon centered, 24px, color `--color-accent` when unlocked, `--color-text-muted` + `grayscale` when locked.
- Right column: `flex: 1`, `display: flex`, `flex-direction: column`, gap `--spacing-xs` (4px).
- Title: 16px semibold, color `--color-text`.
- Description: 14px normal, color `--color-text-muted`.
- Bottom row (LEVEL + progress): `display: flex`, `align-items: center`, gap `--spacing-sm` (8px), `margin-top: --spacing-xs`.
- LEVEL label: 12px semibold uppercase letter-spacing 0.05em. Color `--color-accent` when unlocked, `--color-text-muted` when locked.
- Progress bar: `flex: 1`, height 6px, `border-radius: 3px`. Track: `--color-border`. Fill: `--color-accent` (unlocked) or `--color-border` at 0% (locked).
- Counter (e.g. "2/3"): 12px semibold, color `--color-text-muted`, `white-space: nowrap`.

### AchievementBadgeMini (end-of-game modal)

```
┌──────────────────────────────────────────┐
│  [icon 20px]  Title (semibold 14px)      │  ← border-left: 3px solid {accent/success}
└──────────────────────────────────────────┘
```

- Layout: `flex-direction: row`, align-items center, gap `--spacing-sm` (8px).
- Padding: `--spacing-sm` vertical, `--spacing-md` horizontal.
- Left border: 3px solid `--color-accent` for new unlock (tier 1). 3px solid `--color-success` for upgrade (tier 2+).
- Background: `--color-surface`.
- Border-radius: `--border-radius` (8px).
- Icon: Lucide at 20px. Color `--color-accent` (new) or `--color-success` (upgrade).
- Tier badge: small pill "NIVEL N" at 11px semibold, `--color-accent` / `--color-success` accordingly.

### AchievementModal (end-of-game)

- Reuses `Modal` component with title "Logros desbloqueados".
- Content structure: grouped by player. Player name as section header (14px semibold uppercase, `--color-text-muted`), followed by vertical list of `AchievementBadgeMini` with gap `--spacing-sm`.
- Multiple players separated by `--spacing-md` gap.
- Appears automatically when `achievements_unlocked` list in game response is non-empty.

### TabBar (PlayerProfile)

- Horizontal row of 3 tab buttons: "Stats", "Records", "Logros".
- Full-width, each tab `flex: 1`.
- Height: 44px (touch target).
- Background: `--color-surface`.
- Border-bottom: 1px solid `--color-border`.
- Active tab: border-bottom 2px solid `--color-accent`, color `--color-text`, weight semibold.
- Inactive tab: color `--color-text-muted`, weight normal.
- No hover animation — use `--transition` (0.2s ease) on color only.
- Tab bar position: immediately below the player name header, sticky if content scrolls.

### AchievementCatalog page (`/achievements`)

- Full-page layout mirroring PlayerProfile structure: header with back link + title, main content area.
- List of AchievementCard components in vertical stack, gap `--spacing-sm`.
- Each catalog card shows: same layout as AchievementCard but with holders info instead of personal progress.
- Holders info line: "NIVEL {max_tier} — {n} jugadores" at 12px semibold, `--color-text-muted`.
- Clicking a card opens a `Modal` with title "{achievement title}" and list of holders: player name + tier + date, each row as a list item.

## Interaction States

### Tab switching (PlayerProfile)
- Immediate tab content swap, no animation. Active tab updates border and text color via CSS class toggle.
- Selected tab stored in `useState<'stats' | 'records' | 'logros'>` local to PlayerProfile, default `'stats'`.

### Loading state
- Each tab section shows `<Spinner />` independently. Tab switching does not re-fetch if data already loaded.
- AchievementCatalog page shows `<Spinner />` centered while fetching.

### Empty state — Logros tab (player has no achievements yet)
- Show empty message (see Copywriting section below).

### Empty state — AchievementModal
- Modal does not render at all if no achievements unlocked (D-02). No empty modal.

### Catalog card click
- Opens Modal. No router navigation.

## Icon Mapping

Lucide icon names to be used as `fallback_icon` values in achievement definitions (D-23). Icons rendered at 24px in AchievementCard, 20px in AchievementBadgeMini.

| fallback_icon string | Lucide Component | Achievement use-case |
|---------------------|-----------------|---------------------|
| `"trophy"` | `Trophy` | Victory-related achievements |
| `"flame"` | `Flame` | Win streak achievements |
| `"map"` | `Map` | All-maps achievements |
| `"gamepad-2"` | `Gamepad2` | General gameplay achievements |
| `"star"` | `Star` | Score/performance achievements |
| `"zap"` | `Zap` | Speed or intensity achievements |

`AchievementIcon` component interface:
```
interface AchievementIconProps {
  fallback_icon: string   // one of the keys above
  size?: number           // defaults to 24
  unlocked: boolean       // controls color vs grayscale
}
```
