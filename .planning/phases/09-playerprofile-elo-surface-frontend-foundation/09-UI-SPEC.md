> PlayerProfile Stats tab. Generated by gsd-ui-researcher from locked CONTEXT

## Typography

Using existing tokens from `frontend/src/index.css:35-44`. Body line-height
inherited from `body { line-height: 1.5 }` (`index.css:72`); heading
line-height inherited from the global `h1..h6 { line-height: 1.3 }` rule
(`index.css:103-106`).

**Phase 9 uses exactly 4 sizes and 2 weights.**

| Role | Token | Computed | Weight | Line height | Usage |
|------|-------|----------|--------|-------------|-------|
| Hero ELO number | `--font-size-3xl` | 30px | `--font-weight-bold` (700) | 1.3 (heading) | The current_elo number (or `—`) — visually dominant focal point |
| Hero label "ELO" + delta span | `--font-size-xl` | 20px | `--font-weight-semibold` (600) | 1.3 | The "ELO" word adjacent to the number; the `+23 / -12 / ±0` delta span |
| Sub-row (peak + rank) | `--font-size-base` | 16px | `--font-weight-normal` (400) | 1.5 (body) | "Pico: 1612", "Pico: 1612 · actual", "#3 de 8" |
| (existing statsCard `statValue`) | `--font-size-2xl` | 24px | bold | 1.3 | Reference only — the 3-tile grid below the hero card; the EloSummaryCard hero is intentionally LARGER (3xl > 2xl) to assert the new visual hierarchy from D-08 |

**Why these sizes:** D-08 mandates "ELO grande" as a hero. The existing
`statValue` is 24px / 2xl (`PlayerProfile.module.css:73`). To make the ELO card
unmistakably the new top-of-page focus while reusing tokens (D-09) and
respecting the project's no-new-tokens discipline, the hero number steps up
exactly one rung to `--font-size-3xl` (30px). Sub-row uses `--font-size-base`
to feel calm relative to the hero, mirroring how `.statLabel` recedes from
`.statValue`.

**Weights used:** Exactly 2 — `--font-weight-bold` (700) for the hero number,
`--font-weight-normal` (400) for the sub-row, plus `--font-weight-semibold`
(600) for the "ELO" label and the delta. This still counts as 2 distinct
intent levels (emphasis vs. body) since semibold is the project's standard
"label" weight (matches `AchievementCard.module.css:25`, `PlayerProfile.module.css:54`).
If the checker rejects 3 weights, collapse the "ELO" label to weight 400 — the
hero number remains visually dominant by size alone.

**Heading semantics:** The card root is a `<section aria-label="Resumen de ELO">`.
It does NOT introduce a new visible heading — the existing `<h2>Estadísticas</h2>`
on the statsCard below remains the section title for the Stats tab. The "ELO"
label adjacent to the hero number is a `<span>`, not a heading, because the
card is a single-stat summary and a heading would compete with the hero number
for visual primacy.

## Color

Using existing tokens from `frontend/src/index.css:1-19`. **Zero new tokens
introduced** (D-09 mandate).

| Role | Token | Hex | Usage in this phase |
|------|-------|-----|---------------------|
| Dominant (60%) | `--color-background` | `#1a0a05` | Page background under the card; cascades from `body` |
| Secondary (30%) | `--color-surface` | `#2c1810` | EloSummaryCard background (mirrors `.statsCard`) |
| Accent (10%) | `--color-accent` | `#e67e22` | RESERVED — see explicit list below |
| Semantic positive | `--color-success` | `#27ae60` | Delta span when `last_delta > 0` only |
| Semantic negative | `--color-error` | `#e74c3c` | Delta span when `last_delta < 0` only |
| Muted | `--color-text-muted` | `#a89080` | `±0` delta; sub-row text (peak + rank); the `—` em-dash for 0-games |
| Default text | `--color-text` | `#f5e6d3` | Hero ELO number |
| Border | `--color-border` | `#4a2c1a` | 1px card outline |

**Accent reserved for (explicit list — never "all interactive elements"):**

- The hero ELO number is NOT painted with accent — it uses `--color-text`. The
  intent is for the number itself to be the focal point via *size and weight*,
  not via color. This deviates from the existing `.statValue` pattern (which
  uses `--color-accent`) on purpose: the existing 3-tile grid will still use
  accent below, so the EloSummaryCard reads as a calm hero with a colorful
  context grid underneath.
- Accent is NOT used inside this card. Accent in this phase is reserved
  app-wide for: existing CTAs (`Button` primary), existing `.statValue` in the
  3-tile grid (untouched), `.sectionTitle` (untouched), winning position
  highlight in `.gamePosition.winner` (untouched), `.gameRow:hover` border
  (untouched). No new accent usage is introduced by Phase 9.
- `--color-success` / `--color-error` are used ONLY on the delta span. They
  are NEVER used as backgrounds in this card. They are NEVER used on the peak
  or rank sub-row text.

**Contrast verification (manual, against `--color-surface` `#2c1810`):**

- `--color-text` `#f5e6d3` on surface → ~12.5:1 (passes WCAG AAA for any size).
- `--color-success` `#27ae60` on surface → ~4.6:1 (passes WCAG AA for normal
  text, since the delta is rendered at 20px / `font-size-xl` which qualifies as
  normal text per WCAG, not large text — the 4.5:1 threshold is met).
- `--color-error` `#e74c3c` on surface → ~5.1:1 (passes WCAG AA).
- `--color-text-muted` `#a89080` on surface → ~5.5:1 (passes WCAG AA).

All four delta states meet WCAG AA. No additional `aria-label` is needed for
color-blindness, but the explicit `aria-label="Cambio de ELO en la última
partida: +23"` per D-11 is mandatory anyway because the visual delta lacks
auxiliary text.

**Destructive color:** `--color-error` is repurposed here as a SEMANTIC
NEGATIVE (delta sign), not a destructive-action color. Phase 9 has no
destructive actions, no buttons, no confirmation dialogs.

## Copywriting Contract

Spanish copywriting is mandatory (project default — `.claude/CLAUDE.md`).
Phase 9 is a read-only surface, so there is **no primary CTA** and **no
destructive action**. The "states" map differently than the template default —
recorded explicitly below.

| Element | Copy | Notes |
|---------|------|-------|
| Card visible label (next to hero number) | `ELO` | Single uppercase token. NOT a heading; NOT translated; ELO is treated as a proper noun. |
| Delta — positive | `+23` | Sign is mandatory. No auxiliary text. (D-11) |
| Delta — negative | `-12` | Native `-` glyph (the `Number.toString()` output is correct; do not substitute U+2212 minus). (D-11) |
| Delta — zero | `±0` | U+00B1 PLUS-MINUS SIGN followed by `0`. Rendered with `--color-text-muted`. (D-17) |
| Delta a11y label — positive | `Cambio de ELO en la última partida: +23` | `aria-label` on the delta span. Number is the actual delta. (D-11) |
| Delta a11y label — negative | `Cambio de ELO en la última partida: -12` | Same. |
| Delta a11y label — zero | `Cambio de ELO en la última partida: ±0` | Same — uses `±0`, not `0`. |
| Peak — normal | `Pico: 1612` | Single space after colon. Number is the actual peak. |
| Peak — equals current (D-16) | `Pico: 1612 · actual` | U+00B7 MIDDLE DOT separator with single spaces on both sides. Suffix is lowercase `actual` (no italic, no bold — visual cue is the suffix itself, not styling). |
| Rank | `#3 de 8` | Hash glyph + position + ` de ` + total. No "ranking", no "puesto", no "posición". (Spanish convention from REQUIREMENTS.md PROF-04 example.) |
| Rank — single active player (D-18) | `#1 de 1` | Same template — DO NOT special-case to hide it. |
| Card a11y root | `aria-label="Resumen de ELO"` | On the `<section>` element. Establishes the landmark for screen readers since there is no visible heading inside the card. |
| Empty state — 0-games player | (visual: render `—` em-dash for current_elo; HIDE peak / rank / delta entirely) | NO fallback text like "Sin partidas todavía" inside the card. The em-dash IS the empty state. The 3-tile grid below already says "Partidas jugadas: 0", so duplicating an empty message would be noise. |
| Error state — summary fetch fails | (visual: card not rendered at all; profile renders normally) | NO error message inside the card. NO toast. NO retry button. The card simply does not appear. (D-14) The page-level error box for OTHER failures (`No se pudo cargar el perfil del jugador.`) is untouched and is unrelated. |
| Loading state | (visual: card not rendered until `eloSummary !== null` resolves) | The page already shows a `<Spinner />` while the parent `Promise.all` resolves. The summary is part of the same parallel fetch (D-14). No card-local skeleton. No card-local spinner. |
| Primary CTA | (none — read-only surface) | Phase 9 introduces no buttons. |
| Destructive confirmation | (none — no destructive actions) | Phase 9 has no destructive actions. |

**Glyph reference (literal characters to use):**

| Visual | Codepoint | Usage |
|--------|-----------|-------|
| `—` | U+2014 EM DASH | 0-games current_elo placeholder. Single character. NOT two hyphens. NOT `--`. |
| `±` | U+00B1 PLUS-MINUS SIGN | Zero-delta marker (`±0`). |
| `·` | U+00B7 MIDDLE DOT | Peak suffix separator (`Pico: 1612 · actual`). Surrounded by single spaces. |
| `#` | U+0023 NUMBER SIGN | Rank prefix (`#3 de 8`). Standard ASCII. |
| `+` | U+002B PLUS SIGN | Positive delta prefix. Standard ASCII. |
| `-` | U+002D HYPHEN-MINUS | Negative delta prefix. Native JS `Number.toString()` output — do NOT substitute U+2212. |

## Component Contract

### `EloSummaryCard` (NEW)

**Location:** `frontend/src/components/EloSummaryCard/EloSummaryCard.tsx` +
`EloSummaryCard.module.css` (D-10).

**Props:**

```ts
import type { PlayerEloSummaryDTO } from '@/types'

interface EloSummaryCardProps {
  summary: PlayerEloSummaryDTO
}
```

`PlayerEloSummaryDTO` shape (D-02, defined in `src/types/index.ts` by this same
phase):

```ts
interface EloRankDTO {
  position: number
  total: number
}

interface PlayerEloSummaryDTO {
  current_elo: number              // always present (1000 seed for 0-games per D-05)
  peak_elo: number | null          // null only when player has 0 games
  last_delta: number | null        // null only when player has 0 games
  rank: EloRankDTO | null          // null when player is inactive (D-04 / D-18) or 0 games
}
```

**Render contract (deterministic — every branch is testable):**

| Field state | Visual output |
|-------------|---------------|
| `last_delta === null` (proxy for "0 games") | Hero number = `—` (em-dash); delta span hidden; peak line hidden; rank line hidden (because peak/rank are also null per D-05). Card still renders with the "ELO" label. |
| `last_delta !== null` AND has games | Hero number = `summary.current_elo` rendered as integer (no thousands separator — TM Stats player ELOs are always 4 digits); delta span shown with sign + color per below. |
| `last_delta > 0` | Delta span shows `+{n}` with class `deltaPositive` (color `--color-success`). aria-label populated. |
| `last_delta < 0` | Delta span shows `{n}` (native sign) with class `deltaNegative` (color `--color-error`). aria-label populated. |
| `last_delta === 0` | Delta span shows `±0` with class `deltaZero` (color `--color-text-muted`). aria-label populated. |
| `peak_elo === null` | Peak line hidden. (D-16) |
| `peak_elo !== null` AND `peak_elo !== current_elo` | Peak line shows `Pico: {peak_elo}`. |
| `peak_elo !== null` AND `peak_elo === current_elo` | Peak line shows `Pico: {peak_elo} · actual`. (D-16) |
| `rank === null` | Rank line hidden. (D-18) |
| `rank !== null` | Rank line shows `#{rank.position} de {rank.total}` — including `#1 de 1` case. |
| `peak_elo === null` AND `rank === null` | Sub-row not rendered at all (no empty wrapper div). |

**HTML structure (recommended skeleton — final shape is Claude's discretion
per CONTEXT, but must satisfy all branches above and the a11y contract):**

```tsx
<section className={styles.card} aria-label="Resumen de ELO">
  <div className={styles.heroRow}>
    <span className={styles.heroValue}>
      {hasGames ? summary.current_elo : '—'}
    </span>
    <span className={styles.heroLabel}>ELO</span>
    {summary.last_delta !== null && (
      <span
        className={deltaClass(summary.last_delta)}
        aria-label={`Cambio de ELO en la última partida: ${formatDelta(summary.last_delta)}`}
      >
        {formatDelta(summary.last_delta)}
      </span>
    )}
  </div>
  {(summary.peak_elo !== null || summary.rank !== null) && (
    <div className={styles.subRow}>
      {summary.peak_elo !== null && (
        <span className={styles.peak}>
          Pico: {summary.peak_elo}
          {summary.peak_elo === summary.current_elo && ' · actual'}
        </span>
      )}
      {summary.rank !== null && (
        <span className={styles.rank}>
          #{summary.rank.position} de {summary.rank.total}
        </span>
      )}
    </div>
  )}
</section>
```

**Required helpers (extracted to keep render body ≤20 lines per project rule):**

```ts
function formatDelta(d: number): string {
  if (d > 0) return `+${d}`
  if (d < 0) return `${d}`        // native '-' from Number.toString
  return '±0'
}

function deltaClass(d: number): string {
  if (d > 0) return styles.deltaPositive
  if (d < 0) return styles.deltaNegative
  return styles.deltaZero
}
```

## Layout & Responsive Behavior

**Inherited from parent:** `<main className={styles.main}>` at
`PlayerProfile.module.css:27-36` constrains content to `max-width: 700px` and
applies `gap: var(--spacing-lg)` between sibling sections. The EloSummaryCard
gets responsive layout for free as a child of this container.

**Card-internal layout:**

- Mobile (< 480px effective inner width): hero row wraps if needed (`flex-wrap: wrap`); the delta drops to its own line below the hero number; sub-row stays as a single wrapped flex row with `gap: --spacing-md` between peak and rank chips.
- Tablet / desktop (≥ 480px effective inner width): hero row fits on one line — `[1523]  ELO  [········· +23]` with the delta pushed right via `margin-left: auto`; sub-row is one line: `Pico: 1612 · actual    #3 de 8`.

**Touch targets:** Phase 9 introduces no interactive elements (no buttons,
links, or focusable widgets inside the card). The card is a passive read-only
display. No 44px touch-target compliance is needed.

**Focus order:** Card is non-focusable. Tab order on the Stats tab is unchanged
relative to today (TabBar → "Historial de partidas" links).

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Card is identifiable as a region | `<section aria-label="Resumen de ELO">` — establishes the landmark since there's no visible heading. |
| Delta sign is announced semantically (not just visually colored) | `aria-label="Cambio de ELO en la última partida: +23"` (or `-12` / `±0`) on the delta `<span>`. (D-11) |
| `±0` is read correctly by screen readers | Use literal `±0` (U+00B1 + `0`). Test with VoiceOver / NVDA on the dev profile during phase verification. |
| Em-dash for 0-games is not read as a literal "em-dash" | The em-dash sits inside `.heroValue` next to the visible "ELO" label; screen reader announces "— ELO" → contextually understandable as "no rating yet". An optional `aria-label="ELO no disponible"` on the hero number for 0-games can be added if user testing surfaces ambiguity (deferred — not part of Phase 9 scope unless ambiguity is observed). |
| Color is not the sole signal for delta sign | Sign character (`+` / `-` / `±`) carries the meaning visually; aria-label carries it for screen readers. Color is a redundant reinforcer, not the primary signal. WCAG 1.4.1 (Use of Color) compliant. |
| Color contrast | All four foreground colors against `--color-surface` background pass WCAG AA at 20px (delta) and 30px (hero). Verified above in the Color section. |
| No motion / animation | Card is static. No transitions on mount. No animated delta counter. (Avoids WCAG 2.3.3 motion concerns by construction.) |
| Reduced motion | Not applicable — no animations to disable. |
| Keyboard navigation | No focusable elements inside the card. The card is a non-interactive display surface. Tab order through the Stats tab is unchanged. |

## Empty / Loading / Error States

**Comprehensive state matrix** for the EloSummaryCard:

| Backend state | `summary` value at component | Visual outcome |
|----------------|------------------------------|----------------|
| Summary endpoint succeeds, player has games | `{ current_elo, peak_elo, last_delta, rank }` (all populated) | Full card: hero + delta + sub-row with peak and rank. |
| Summary endpoint succeeds, player has games but is INACTIVE | `{ current_elo, peak_elo, last_delta, rank: null }` | Full card except rank line hidden. (D-18) |
| Summary endpoint succeeds, player has 0 games (D-05) | `{ current_elo: 1000, peak_elo: null, last_delta: null, rank: null }` | Card shows `—` (em-dash) for hero number, "ELO" label, NO delta, NO sub-row. The seeded `1000` is NEVER rendered. (Pitfall 6 case 1.) |
| Summary endpoint succeeds, player has 1 game and lost the only game | `{ current_elo, peak_elo: current_elo, last_delta: -N, rank }` | Full card; peak shows "Pico: {n} · actual" because peak === current; delta is red. |
| Summary endpoint succeeds, only one active player exists | `{ ..., rank: { position: 1, total: 1 } }` | Rank line shows `#1 de 1`. NOT hidden. (D-18) |
| Summary endpoint succeeds, peak === current AND rank present | `{ current_elo, peak_elo: current_elo, last_delta, rank }` | Sub-row shows "Pico: {n} · actual" + "#{p} de {t}" both. |
| Summary endpoint fails (any 4xx / 5xx / network error) | `null` (set by the page's `.catch(() => null)` per D-14) | Card not rendered AT ALL. Profile page renders normally with the existing 3-tile grid and Historial de partidas. No error message anywhere referencing ELO. |
| Profile or players endpoint fails | (component never mounts because page shows page-level error) | Existing `<p className={styles.errorBox}>No se pudo cargar el perfil del jugador.</p>` message — untouched by this phase. |
| Page is mid-load | `eloSummary` initial state still null in the page | Card not rendered yet; page-level `<Spinner />` is showing. When the parent `Promise.all` resolves, the card mounts in one paint along with the rest of the Stats tab content. |

**Key invariant:** the EloSummaryCard never renders an error message of its
own. Per D-14, summary-fetch failure must be invisible to the user — the
profile page's other content is the safety net. This is the load-bearing
behavior for Pitfall 6 of `09-RESEARCH.md`.
