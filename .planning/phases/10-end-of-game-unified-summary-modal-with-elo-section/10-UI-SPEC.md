## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--spacing-xs`) | Icon gaps, close-button padding, badge inline padding |
| sm | 8px (`--spacing-sm`) | Row gaps inside ELO list, badge list gaps, compact element spacing |
| md | 16px (`--spacing-md`) | Default column gap in ELO rows, section internal padding, grid column gaps |
| lg | 24px (`--spacing-lg`) | Section-to-section separation inside modal, modal header bottom margin |
| xl | 32px (`--spacing-xl`) | Modal outer padding, footer top margin |
| 2xl | 48px (`--spacing-2xl`) | Not used in this phase |

Exceptions: Modal overlay padding is 16px (`--spacing-md`) on mobile (matches existing `Modal.module.css`). Touch targets for the close button must be at minimum 44px — achieved via `padding: var(--spacing-xs)` on a large `font-size` glyph (existing pattern, no change needed).

Source: `frontend/src/index.css` tokens; `frontend/src/components/Modal/Modal.module.css` (existing overlay/modal layout).

## Layout Contract

### Modal structure

```
Modal (base — 480px max-width, max-height 90vh, overflow-y auto)
  └── h2.title  "Resumen de partida"
  └── Section: Resultados
      └── h3.sectionTitle  "Resultados"
      └── p.gameMeta       {formatted date}
      └── div.rankingList
          └── div.rankRow × N   [grid: 48px | 1fr | auto | auto]
               position | playerName | totalPts | mcTotal
  └── Section: Records
      └── h3.sectionTitle  "Records"
      └── RecordsSection   (existing component)
  └── Section: Logros
      └── h3.sectionTitle  "Logros"
      └── AchievementsSection (groups by player → AchievementBadgeMini)
  └── Section: ELO
      └── h3.sectionTitle  "ELO"
      └── div.eloList
          └── div.eloRow × N   [grid: 48px | 1fr | auto | auto]
               position | playerName | eloPair | delta
  └── footer
      └── Button variant="primary"  "Continuar"
```

### ELO row grid

```
grid-template-columns: 48px 1fr auto auto
```

- Column 1 (48px fixed): Position label — `#1`, `#2`, … — `--font-weight-bold`, `--font-size-lg`, color `--color-text`
- Column 2 (flex): Player name — `--font-weight-medium`, `--font-size-base`, color `--color-text`
- Column 3 (auto): ELO pair — `elo_before → elo_after` — `--font-size-sm`, `--font-weight-normal`, color `--color-text-muted`
- Column 4 (auto): Delta — `+23` / `-12` / `±0` — `--font-size-base`, `--font-weight-normal`, color by sign (success/error/muted)

ELO row renders inside `--color-background` background, `1px solid var(--color-border)` border, `--border-radius` (8px), `padding: var(--spacing-sm) var(--spacing-md)`.

Gap between rows: `var(--spacing-sm)` (8px).

Arrow character between elo_before and elo_after: `→` (U+2192) — plain Unicode, no icon library.

Source: CONTEXT.md D-06, D-07, D-08; GameRecords `.rankRow` grid pattern (adapted to 4 columns).

## States and Interactions

### Modal open/close

| Trigger | Behavior |
|---------|----------|
| `GameRecords` mounts | Modal opens immediately (always — D-03) |
| Click outside modal (overlay) | Modal closes |
| Press `Escape` | Modal closes |
| Click "Continuar" button | Modal closes |
| After close | Modal is gone; `GameRecords` shows only header + "Volver al inicio" button |

No "reopen" mechanism — D-03 explicitly locks this.

### Loading

The modal opens as soon as `GameRecords` mounts, before all fetches resolve. Sections that have not yet loaded render a `<Spinner />` in place of their content. Once a fetch resolves, content replaces the spinner.

Implementation note (Claude's Discretion per CONTEXT.md): Fetches run in parallel (not Promise.all-blocking). Each section manages its own loading state independently for progressive display.

### Section empty states

| Section | Empty condition | Display |
|---------|----------------|---------|
| Resultados | Always has data (game has ≥2 players) | N/A |
| Records | `records` array is empty or all `achieved: false` | "Ningún record nuevo en esta partida." — `--font-size-sm`, `--color-text-muted` |
| Logros | All player achievement lists are empty | "Ningún logro desbloqueado." — `--font-size-sm`, `--color-text-muted` |
| ELO | `fetchEloChanges` fails after 1 retry | Section omitted entirely (no error message shown to user); `console.warn` logged |
| ELO | Always has data when fetch succeeds (ELO fires on every game) | N/A |

Source: CONTEXT.md D-04, D-05.

### GameRecords page (post-modal)

Once the modal is closed, `GameRecords.tsx` renders:

```
div.page
  div.card (max-width 600px, centered)
    div.header
      span.icon  🏆
      h1.title   "¡Partida guardada!"
    div.actions (centered)
      Button  "Volver al inicio"
```

No inline sections (Resultados, Records) — those live exclusively in the modal now (D-01).

## Accessibility

- Modal overlay: `aria-modal="true"` or equivalent focus trap; Escape key handler exists in `Modal.tsx` — no change needed.
- Close button: `aria-label="Cerrar"` (existing `Modal.tsx` pattern).
- ELO section: wrap in `<section aria-label="Cambios de ELO">` to give screen readers a landmark.
- Delta values: `aria-label` on each delta `<span>` — e.g., `aria-label="Cambio de ELO: +23"` (mirrors `EloSummaryCard` pattern from Phase 9).
- Position column: `aria-label="Posición 1"` on position `<span>` (e.g., for `#1`).
- Color-coded deltas (success/error/muted) must not rely on color alone — the sign character (`+`, `-`, `±`) provides the semantic distinction.
