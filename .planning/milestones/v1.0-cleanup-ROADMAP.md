**Key Decisions:**

- **Phase 5 strategy: option (a) from audit §10.** Caller `GameRecords.tsx` consumes the existing `useGames.fetchAchievements` hook (which already implements D-09/D-10 retry); `useGames.ts` and `api/achievements.ts` are not modified. Preserves the documented retry contract instead of duplicating it.
- **Phase 5 human-verify checkpoint substituted with automated tests.** Plan 05-02 Task 2 (`checkpoint:human-verify`) was converted to 9 vitest cases (4 hook + 5 component integration) plus an end-to-end backend smoke. Manual cases A/B/C/D remain available but no longer block the phase. Substitution explicitly pre-approved by the user.
- **Phase 6 props removed, not implemented.** `is_upgrade` and `max_tier` were never wired in the component bodies — they were declared but unused. Removing them is the simpler resolution; transport DTOs (`AchievementUnlockedDTO.is_upgrade`, `PlayerAchievementDTO.max_tier`) stay intact because they're still part of the API contract consumed by other callers.
- **Phase 6 cosmetic blank-line cleanup hand-edited via Python script, not formatter.** Project doesn't ship ruff/black/autopep8/flake8 — adding one was out of scope. Idempotent, AST-equivalent diff.
- **Phase 7 was inline-executed.** Planner agent hit usage limit; planning was done manually given the scope was 100% specified by the audit. Saved tokens, no quality loss (verifier passed 8/8).

**Issues Resolved:**

- Singleton pattern violation in services layer (high severity audit gap)
- Orphan Phase 02 retry contract restored to live UI path
- 4 dead-code surface items cleaned (DTO field, 2 props, blank lines)
- 8 SUMMARYs newly traceable by grep on top-level `requirements:` field
- Doc drift "5 evaluators" corrected to actual "12 evaluators" (registry.py state)
