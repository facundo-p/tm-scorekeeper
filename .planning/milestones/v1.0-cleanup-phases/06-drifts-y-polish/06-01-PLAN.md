<objective>
Eliminar la propiedad `title` de `AchievementCatalogItemDTO` en el backend (schema + mapper) y en el frontend (interfaz TypeScript). El audit `v1.0-MILESTONE-AUDIT.md` §4 (drift de severidad **low**, Phase 02) detecta que este campo se está poblando con `definition.description` en `mappers/achievement_mapper.py:82`, dejando `title` y `description` con el mismo valor. El frontend ya se adaptó renderizando `item.description`, así que `item.title` es redundante y engañoso.

**Por qué se elimina (no se rellena con un valor distinto):** El orquestador ya verificó que `frontend/src/pages/AchievementCatalog/AchievementCatalog.tsx` no consume `item.title` en ningún punto — renderiza `item.description` y, dentro del modal, `tier.title` (que pertenece a `AchievementTierInfoDTO`, otro DTO no afectado). No existe en el registry actual una fuente para un título humano distinto del `description`. Crear uno sería invención sin source. La opción correcta del audit (parénthesis "or the field is removed") es eliminar.

**Output:** DTO backend sin `title`, mapper sin la asignación, interfaz TypeScript sin la propiedad, contrato HTTP `GET /achievements/catalog` con un campo menos en cada item (cambio compatible para el único consumidor que ya ignora ese campo). Tests backend y frontend verdes.
</objective>

    tiers = [
        AchievementTierInfoDTO(level=t.level, threshold=t.threshold, title=t.title)

        code=evaluator.code,
        title=evaluator.definition.description,    # ← ESTA LÍNEA SE ELIMINA
        description=evaluator.definition.description,
        icon=evaluator.definition.icon,
